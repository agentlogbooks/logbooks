# Ideation Logbook

Per-topic multi-entity SQLite database holding the evolving state of ideas across sessions. The authoritative working memory of the `ideation` skill: framing context, the ideas themselves, their lineage, per-metric assessments, and a full operator-run audit log.

## Physical stores

- **Authoritative:** SQLite at `./.ideation/{slug}/logbook.sqlite` — seven tables covering context (`frames`, `facts`), ideas (`ideas`, `lineage`, `assessments`), topic self-description (`topic_meta`), and cross-cutting audit (`operator_runs`).
- **Projection: reports** — role: `export-only` — markdown files at `./.ideation/{slug}/reports/<run_id>-<kind>.md` — regenerated by `decide.compare` / `decide.converge` / `decide.export`; never edited.
- **Projection: session summaries** — role: `export-only` — optional `./.ideation/{slug}/runs/<run_id>/summary.md` — human-readable recap generated at session end.

Config side-cars (not projections, not logbook state): `./.ideation/{slug}/criteria-<run_id>.json` — one file per `evaluate.criteria` run, holds criteria + weights used by the matching `evaluate.score` run.

No JSONL run-trace projection. Event-level audit lives relationally in `operator_runs`.

## Design principles

- **Ideas are first-class**; operators write atoms (new ideas, lineage edges, facts, assessments) and tag every write with `operator_run_id` for full provenance.
- **Context is separated from ideas.** `frames` and `facts` hold problem-space state; ideas reference a `frame_id_at_birth` so reframing doesn't orphan earlier ideas.
- **Assessments are rows, not columns.** A new criterion (`feasibility`, `web_stress_verdict`) is a new `metric` value, not a new column. The "current" assessment for `(idea_id, metric)` is the latest by `created_at`. This solves the column-drift problem persistent logbooks would otherwise hit.
- **Lineage is a graph, not free text.** `lineage` is n-to-n; a recursive CTE traverses ancestors or descendants.
- **Operator runs are the action log.** Every operator call writes one row; `cohort_ids` JSON snapshots which ideas were touched. "What operators have run on idea 17?" is a single query.

## Schema

### topic_meta

Singleton row describing this file. Self-describes the topic so an instance moved to another path is still identifiable.

| Column | Type | Required | Semantics |
|---|---|---|---|
| `topic_slug` | TEXT PRIMARY KEY | yes | Stable slug matching the enclosing directory at creation. Authoritative even if the directory is later renamed. |
| `description` | TEXT | no | One-line topic description. |
| `owner` | TEXT | yes | Creator — git user name or the literal string `personal`. |
| `created_at` | TEXT | yes | ISO 8601 creation timestamp. |

### frames

Versioned context objects. Each `frame.discover` or `frame.reframe` inserts a new row; exactly one row has `active=1` at any time.

| Column | Type | Required | Semantics |
|---|---|---|---|
| `frame_id` | INTEGER PRIMARY KEY AUTOINCREMENT | yes | Sequential id. |
| `version` | INTEGER | yes | 1 on first insert, incremented on each reframe. |
| `active` | INTEGER CHECK (active IN (0,1)) | yes | Exactly one active per topic at any time; enforced in `ideation_db.py add-frame` (transaction flips prior active to 0). |
| `problem_statement` | TEXT | yes | One-sentence statement of the user's intent this frame serves. |
| `root_causes` | TEXT | yes | JSON array of root-cause strings. |
| `hmw_questions` | TEXT | yes | JSON array of "How might we…" questions. |
| `triz_contradiction` | TEXT | no | JSON object `{improve: "...", worsen: "..."}` — may be absent if no trade-off identified. |
| `ifr_statement` | TEXT | no | Ideal Final Result — single sentence. |
| `supersedes_frame_id` | INTEGER | no | FK to `frames.frame_id`; non-null when this row replaces a prior active frame. |
| `operator_run_id` | INTEGER | yes | FK to `operator_runs.operator_run_id`; records which operator produced this frame. |
| `created_at` | TEXT | yes | ISO 8601. |

### facts

Citable context facts harvested from the web or submitted by operators. Supports, refutes, or stays neutral on the problem space.

| Column | Type | Required | Semantics |
|---|---|---|---|
| `fact_id` | INTEGER PRIMARY KEY AUTOINCREMENT | yes | Sequential id. |
| `claim` | TEXT | yes | 1–2 sentence citable fact. |
| `source_url` | TEXT | no | URL where the claim was observed; NULL for operator-contributed internal claims. |
| `source_label` | TEXT | no | Human-readable source (e.g. "Nielsen 2024 survey"). |
| `confidence` | TEXT CHECK (confidence IN ('strong','medium','weak')) | yes | Epistemic confidence at the time of capture. |
| `stance` | TEXT CHECK (stance IN ('supports','adversarial','neutral')) | yes | Relation to the current framing. |
| `operator_run_id` | INTEGER | yes | FK to `operator_runs.operator_run_id`. |
| `created_at` | TEXT | yes | ISO 8601. |

### ideas

The spine. Content fields are immutable after insert; lifecycle fields (`status`, `score_summary`, `evidence_state`, `next_action`, `updated_at`) are patched in place.

| Column | Type | Required | Semantics | Mutable |
|---|---|---|---|---|
| `idea_id` | INTEGER PRIMARY KEY AUTOINCREMENT | yes | Sequential id. | — |
| `title` | TEXT | yes | Short name. | no |
| `description` | TEXT | yes | 2–3 sentences, "coffee-talk" — no jargon, no elaboration. | no |
| `kind` | TEXT CHECK (kind IN ('seed','variant','hybrid','refinement','counter')) | yes | How this idea came into being. | no |
| `tag` | TEXT CHECK (tag IN ('SAFE','BOLD','WILD') OR tag IS NULL) | no | Boldness tag; set by generating operator. | no |
| `temperature_zone` | TEXT CHECK (temperature_zone IN ('FIRE','PLASMA','ICE','GHOST','MIRROR') OR temperature_zone IS NULL) | no | Temperature zone for transform operators that consume zone parameters. | no |
| `status` | TEXT CHECK (status IN ('active','shortlisted','parked','selected','rejected')) | yes | Lifecycle state; default `active` at insert. | **yes** |
| `score_summary` | REAL | no | Latest composite score, denormalized from `assessments` for cheap sorting. Rewritten on each `evaluate.score`. | **yes** |
| `evidence_state` | TEXT CHECK (evidence_state IN ('untested','supported','stressed','disputed') OR evidence_state IS NULL) | no | Summary of the idea's evidence posture after validation operators. | **yes** |
| `next_action` | TEXT | no | Free-text user note for follow-up. | **yes** |
| `origin_operator_run_id` | INTEGER | yes | FK to `operator_runs.operator_run_id`; the run that generated this idea. | no |
| `frame_id_at_birth` | INTEGER | yes | FK to `frames.frame_id`; the active frame when this idea was inserted. Never rewritten on reframe. | no |
| `created_at` | TEXT | yes | ISO 8601. | — |
| `updated_at` | TEXT | yes | ISO 8601; rewritten on any mutable-field patch. | **yes** |

### lineage

Directed parent→child edges between ideas. Many-to-many; one child can have multiple parents (hybrids).

| Column | Type | Required | Semantics |
|---|---|---|---|
| `child_idea_id` | INTEGER | yes | FK to `ideas.idea_id`. |
| `parent_idea_id` | INTEGER | yes | FK to `ideas.idea_id`. |
| `relation` | TEXT CHECK (relation IN ('derived_from','hybrid_of','refinement_of','counter_of')) | yes | Nature of the parent→child edge. |
| `operator_run_id` | INTEGER | yes | FK to `operator_runs.operator_run_id`. |
| PRIMARY KEY | (`child_idea_id`, `parent_idea_id`) | — | Composite. A given child/parent pair appears at most once. |

### assessments

Per-metric judgments on ideas. Append-only; the current assessment for `(idea_id, metric)` is the most recent row by `created_at`.

| Column | Type | Required | Semantics |
|---|---|---|---|
| `assessment_id` | INTEGER PRIMARY KEY AUTOINCREMENT | yes | Sequential id. |
| `idea_id` | INTEGER | yes | FK to `ideas.idea_id`. |
| `metric` | TEXT | yes | Metric name, e.g. `feasibility`, `novelty`, `taste`, `web_stress_verdict`, `six_hats.black`, `brilliance.elegance`, `tension.improve_vs_worsen`. Namespaced with `.` for structured metrics. |
| `value` | TEXT | yes | Metric value. Flexible: stringified number, enum label, or short phrase. |
| `value_numeric` | REAL | no | Parsed numeric value when the metric is scalar; enables `ORDER BY value_numeric`. |
| `rationale` | TEXT | no | One or two sentences explaining the judgment. |
| `evidence_fact_ids` | TEXT | no | JSON array of `fact_id`s cited in support of the assessment. NULL when no external evidence. |
| `operator_run_id` | INTEGER | yes | FK to `operator_runs.operator_run_id`. |
| `created_at` | TEXT | yes | ISO 8601. |

### operator_runs

The cross-cutting action log. One row per operator invocation; the orchestrator inserts at `status=pending`, then patches `status` / `ended_at` / `outcome_summary` / `error` when the operator returns.

| Column | Type | Required | Semantics |
|---|---|---|---|
| `operator_run_id` | INTEGER PRIMARY KEY AUTOINCREMENT | yes | Sequential id. |
| `run_id` | TEXT | yes | UUID identifying one `ideation` invocation. |
| `plan_step_index` | INTEGER | no | 1-based position in the session plan; NULL for ad-hoc operator calls outside a plan. |
| `operator_name` | TEXT | yes | Fully qualified name, e.g. `transform.scamper`. |
| `operator_persona` | TEXT | no | Persona parameter for parameterized operators (e.g. `innovator`). |
| `params` | TEXT | no | JSON object of params passed to the operator. |
| `cohort_ids` | TEXT | yes | JSON array of literal `idea_id`s the operator was run on. Snapshotted at step time so the log records what actually ran, not what was queried. |
| `status` | TEXT CHECK (status IN ('pending','succeeded','failed','skipped')) | yes | Lifecycle state; inserted as `pending`, patched on completion. |
| `outcome_summary` | TEXT | no | Short human-readable recap, e.g. "appended 12 ideas, 3 lineage rows". Written on finalize. |
| `error` | TEXT | no | Error message if `status='failed'`. |
| `user_approved` | INTEGER CHECK (user_approved IN (0,1)) | yes | 0 if user declined a checkpoint that preceded this operator; 1 otherwise. |
| `started_at` | TEXT | yes | ISO 8601. |
| `ended_at` | TEXT | no | ISO 8601; NULL while `status='pending'`. |

## Identity

### topic_meta
Natural key: `topic_slug` is the PK. Single row per file.

### frames
Row key: auto-increment `frame_id`. Uniqueness of the "active" frame is enforced by `ideation_db.py add-frame`, which wraps the insert and the `UPDATE frames SET active=0 WHERE active=1` in one transaction.

### facts
Row key: auto-increment `fact_id`.

### ideas
Row key: auto-increment `idea_id`. No domain fingerprint at v1 — cross-topic dedup is out of scope; within a topic, users can see near-duplicates in `decide.compare`. A `semantic_fingerprint` column may be added later if dedup across sessions becomes load-bearing.

### lineage
Composite primary key: `(child_idea_id, parent_idea_id)`. Enforces that a given parent/child pair appears at most once even if two operator runs would produce the same edge.

### assessments
Row key: auto-increment `assessment_id`. Latest-per-(idea, metric) is a query pattern, not a unique constraint.

### operator_runs
Row key: auto-increment `operator_run_id`. `run_id` (session UUID) scopes rows to one `ideation` invocation; `cohort_ids` JSON snapshots which ideas were touched.

## Partial rows

Convention per record type.

### topic_meta
Nullable fields (`description`) use SQL NULL. `topic_slug`, `owner`, `created_at` are required at insert.

### frames
Nullable fields (`triz_contradiction`, `ifr_statement`, `supersedes_frame_id`) use SQL NULL. All other fields required at insert.

### facts
Nullable fields (`source_url`, `source_label`) use SQL NULL. `confidence` and `stance` are required — no default; the generating operator always picks one.

### ideas
Nullable fields (`tag`, `temperature_zone`, `score_summary`, `evidence_state`, `next_action`) use SQL NULL — never empty string. `status` defaults to `active` at insert.

### lineage
No nullable fields. All columns required.

### assessments
Nullable fields (`value_numeric`, `rationale`, `evidence_fact_ids`) use SQL NULL. The generating operator sets `value_numeric` only when the metric is scalar.

### operator_runs
Nullable fields (`plan_step_index`, `operator_persona`, `params`, `outcome_summary`, `error`, `ended_at`) use SQL NULL. `ended_at` remains NULL while `status='pending'`.

## Corrections

### topic_meta
Patch in place. `description` and `owner` may be edited; `topic_slug` and `created_at` are immutable.

### frames
Append-only. A reframe inserts a new row with incremented `version`, sets `supersedes_frame_id` to the prior active frame, and the accompanying `add-frame` transaction flips the prior frame's `active` to 0. Earlier frames stay readable; ideas born under them keep their `frame_id_at_birth`.

### facts
Append-only. Facts are never patched; a refuting fact is inserted with `stance='adversarial'` and (optionally) a rationale in the generating operator's `outcome_summary`. If a fact's provenance changes, it's a new row, not an edit.

### ideas
Content fields (`title`, `description`, `kind`, `tag`, `temperature_zone`, `origin_operator_run_id`, `frame_id_at_birth`, `created_at`) are immutable after insert. Lifecycle fields (`status`, `score_summary`, `evidence_state`, `next_action`, `updated_at`) are patched in place.

To meaningfully revise an idea, do not edit it — run `transform.refine` which inserts a child with `kind='refinement'` and a `lineage` row linking child→parent with `relation='refinement_of'`. This preserves the edit history as a graph.

### lineage
Append-only. Edges are never deleted. Duplicate edges are prevented by the composite PK.

### assessments
Append-only. A later assessment for the same `(idea_id, metric)` is added as a new row; queries that want "current" read the latest by `created_at`. No `UPDATE` of prior rows under any circumstance — including from the same operator_run_id.

### operator_runs
Insert-then-finalize. Row is inserted with `status='pending'`, `started_at=now`, `ended_at=NULL`. Orchestrator patches `status`, `ended_at`, `outcome_summary`, and (on failure) `error`. No other columns are ever patched; no row is ever deleted.

## SQLite initialization

Run by `ideation_db.py init-topic <slug>` when the user invokes the skill on a new slug:

```bash
sqlite3 ./.ideation/<slug>/logbook.sqlite "
PRAGMA foreign_keys = ON;
PRAGMA journal_mode = WAL;

CREATE TABLE IF NOT EXISTS topic_meta (
  topic_slug TEXT PRIMARY KEY,
  description TEXT,
  owner TEXT NOT NULL,
  created_at TEXT NOT NULL
);

CREATE TABLE IF NOT EXISTS operator_runs (
  operator_run_id INTEGER PRIMARY KEY AUTOINCREMENT,
  run_id TEXT NOT NULL,
  plan_step_index INTEGER,
  operator_name TEXT NOT NULL,
  operator_persona TEXT,
  params TEXT,
  cohort_ids TEXT NOT NULL,
  status TEXT NOT NULL CHECK (status IN ('pending','succeeded','failed','skipped')),
  outcome_summary TEXT,
  error TEXT,
  user_approved INTEGER NOT NULL CHECK (user_approved IN (0,1)),
  started_at TEXT NOT NULL,
  ended_at TEXT
);

CREATE TABLE IF NOT EXISTS frames (
  frame_id INTEGER PRIMARY KEY AUTOINCREMENT,
  version INTEGER NOT NULL,
  active INTEGER NOT NULL CHECK (active IN (0,1)),
  problem_statement TEXT NOT NULL,
  root_causes TEXT NOT NULL,
  hmw_questions TEXT NOT NULL,
  triz_contradiction TEXT,
  ifr_statement TEXT,
  supersedes_frame_id INTEGER REFERENCES frames(frame_id),
  operator_run_id INTEGER NOT NULL REFERENCES operator_runs(operator_run_id),
  created_at TEXT NOT NULL
);

CREATE TABLE IF NOT EXISTS facts (
  fact_id INTEGER PRIMARY KEY AUTOINCREMENT,
  claim TEXT NOT NULL,
  source_url TEXT,
  source_label TEXT,
  confidence TEXT NOT NULL CHECK (confidence IN ('strong','medium','weak')),
  stance TEXT NOT NULL CHECK (stance IN ('supports','adversarial','neutral')),
  operator_run_id INTEGER NOT NULL REFERENCES operator_runs(operator_run_id),
  created_at TEXT NOT NULL
);

CREATE TABLE IF NOT EXISTS ideas (
  idea_id INTEGER PRIMARY KEY AUTOINCREMENT,
  title TEXT NOT NULL,
  description TEXT NOT NULL,
  kind TEXT NOT NULL CHECK (kind IN ('seed','variant','hybrid','refinement','counter')),
  tag TEXT CHECK (tag IN ('SAFE','BOLD','WILD')),
  temperature_zone TEXT CHECK (temperature_zone IN ('FIRE','PLASMA','ICE','GHOST','MIRROR')),
  status TEXT NOT NULL DEFAULT 'active' CHECK (status IN ('active','shortlisted','parked','selected','rejected')),
  score_summary REAL,
  evidence_state TEXT CHECK (evidence_state IN ('untested','supported','stressed','disputed')),
  next_action TEXT,
  origin_operator_run_id INTEGER NOT NULL REFERENCES operator_runs(operator_run_id),
  frame_id_at_birth INTEGER NOT NULL REFERENCES frames(frame_id),
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);

CREATE TABLE IF NOT EXISTS lineage (
  child_idea_id INTEGER NOT NULL REFERENCES ideas(idea_id),
  parent_idea_id INTEGER NOT NULL REFERENCES ideas(idea_id),
  relation TEXT NOT NULL CHECK (relation IN ('derived_from','hybrid_of','refinement_of','counter_of')),
  operator_run_id INTEGER NOT NULL REFERENCES operator_runs(operator_run_id),
  PRIMARY KEY (child_idea_id, parent_idea_id)
);

CREATE TABLE IF NOT EXISTS assessments (
  assessment_id INTEGER PRIMARY KEY AUTOINCREMENT,
  idea_id INTEGER NOT NULL REFERENCES ideas(idea_id),
  metric TEXT NOT NULL,
  value TEXT NOT NULL,
  value_numeric REAL,
  rationale TEXT,
  evidence_fact_ids TEXT,
  operator_run_id INTEGER NOT NULL REFERENCES operator_runs(operator_run_id),
  created_at TEXT NOT NULL
);

CREATE INDEX IF NOT EXISTS idx_ideas_status ON ideas(status);
CREATE INDEX IF NOT EXISTS idx_ideas_kind ON ideas(kind);
CREATE INDEX IF NOT EXISTS idx_ideas_origin_run ON ideas(origin_operator_run_id);
CREATE INDEX IF NOT EXISTS idx_ideas_frame ON ideas(frame_id_at_birth);
CREATE INDEX IF NOT EXISTS idx_assessments_idea ON assessments(idea_id);
CREATE INDEX IF NOT EXISTS idx_assessments_idea_metric_created ON assessments(idea_id, metric, created_at DESC);
CREATE INDEX IF NOT EXISTS idx_lineage_child ON lineage(child_idea_id);
CREATE INDEX IF NOT EXISTS idx_lineage_parent ON lineage(parent_idea_id);
CREATE INDEX IF NOT EXISTS idx_op_runs_run_id ON operator_runs(run_id);
CREATE INDEX IF NOT EXISTS idx_op_runs_operator ON operator_runs(operator_name);
CREATE INDEX IF NOT EXISTS idx_frames_active ON frames(active);
CREATE INDEX IF NOT EXISTS idx_facts_stance ON facts(stance);
CREATE INDEX IF NOT EXISTS idx_facts_confidence ON facts(confidence);
"
```

The DDL is emitted verbatim by `ideation_db.py init-topic`. The `IF NOT EXISTS` clauses make the command safe to re-run on an already-initialized instance.

### Active frame

```bash
sqlite3 ./.ideation/coffee-shop-q2/logbook.sqlite \

   FROM frames
   WHERE active = 1;"
```

### All active ideas

```bash
sqlite3 ./.ideation/coffee-shop-q2/logbook.sqlite \
  "SELECT idea_id, title, description, kind, tag, temperature_zone, score_summary
   FROM ideas
   WHERE status = 'active'
   ORDER BY idea_id;"
```

### Top N ideas by composite score

```bash
sqlite3 ./.ideation/coffee-shop-q2/logbook.sqlite \
  "SELECT idea_id, title, score_summary, evidence_state, tag
   FROM ideas
   WHERE status IN ('active','shortlisted')
     AND score_summary IS NOT NULL
   ORDER BY score_summary DESC
   LIMIT 5;"
```

### Top N ideas by a specific metric (latest assessment per idea)

```bash
sqlite3 ./.ideation/coffee-shop-q2/logbook.sqlite \
  "WITH latest AS (
     SELECT idea_id, value_numeric,
            ROW_NUMBER() OVER (PARTITION BY idea_id ORDER BY created_at DESC) AS rn
     FROM assessments
     WHERE metric = 'taste'
   )
   SELECT i.idea_id, i.title, latest.value_numeric AS taste_score
   FROM ideas i
   JOIN latest ON latest.idea_id = i.idea_id AND latest.rn = 1
   WHERE i.status = 'active'
   ORDER BY latest.value_numeric DESC
   LIMIT 3;"
```

### Children of a specific idea

```bash
sqlite3 ./.ideation/coffee-shop-q2/logbook.sqlite \
  "SELECT i.idea_id, i.title, i.kind, l.relation
   FROM lineage l
   JOIN ideas i ON i.idea_id = l.child_idea_id
   WHERE l.parent_idea_id = 17;"
```

### Full ancestry tree of an idea (recursive CTE)

```bash
sqlite3 ./.ideation/coffee-shop-q2/logbook.sqlite \
  "WITH RECURSIVE ancestry(idea_id, depth) AS (
     SELECT 47, 0
     UNION ALL
     SELECT l.parent_idea_id, a.depth + 1
     FROM lineage l
     JOIN ancestry a ON l.child_idea_id = a.idea_id
   )
   SELECT i.idea_id, i.kind, i.title, a.depth
   FROM ancestry a
   JOIN ideas i ON i.idea_id = a.idea_id
   ORDER BY a.depth;"
```

### Latest assessment per metric for one idea

```bash
sqlite3 ./.ideation/coffee-shop-q2/logbook.sqlite \
  "SELECT metric, value, value_numeric, rationale, created_at
   FROM assessments
   WHERE idea_id = 17
     AND created_at = (
       SELECT MAX(created_at) FROM assessments a2
       WHERE a2.idea_id = assessments.idea_id AND a2.metric = assessments.metric
     )
   ORDER BY metric;"
```

### All operator runs in a session

```bash
sqlite3 ./.ideation/coffee-shop-q2/logbook.sqlite \
  "SELECT operator_run_id, plan_step_index, operator_name, operator_persona,
          cohort_ids, status, outcome_summary
   FROM operator_runs

```

### Which operators have touched idea 17

```bash
sqlite3 ./.ideation/coffee-shop-q2/logbook.sqlite \
  "SELECT operator_run_id, operator_name, operator_persona, status, started_at
```

Ideas involved in the latest `evaluate.tension` run.

```bash
sqlite3 ./.ideation/coffee-shop-q2/logbook.sqlite \
```

## Validation

### Completeness rules per record type

Operators must produce complete rows at insert time. A `NOT NULL` column being present does not mean the content is meaningful; the rules below are the richer readiness checks the skill's validation layer enforces.

**frames** — `root_causes` JSON must decode to an array of at least one string; `hmw_questions` must decode to an array of at least one question. `active=1` is enforced exclusive at the `add-frame` transaction layer.

**facts** — `claim` must be ≤ 300 characters. `stance` must be one of the three allowed values.

**ideas** — `title` must be 3–80 characters; `description` must be 1–4 sentences (heuristic: 40–600 characters, no more than 5 sentence-terminating punctuation marks). `origin_operator_run_id` must reference an `operator_runs` row in this file. `frame_id_at_birth` must reference the `frames` row that was `active=1` at the moment of insert (enforced in `add-idea`).

**lineage** — both `child_idea_id` and `parent_idea_id` must exist. Self-loops (`child = parent`) are rejected.

**assessments** — `metric` must match `^[a-z][a-z0-9_]*(\.[a-z][a-z0-9_]*)*$` (namespaced snake_case). If `value_numeric IS NOT NULL`, `value` must stringify the same number to ±1e-6 tolerance.

**operator_runs** — at finalize, `ended_at` must be ≥ `started_at`. `status='failed'` must have `error IS NOT NULL`. `status='succeeded'` must have `outcome_summary IS NOT NULL`.

### Incomplete-row scan

Find rows that are structurally valid but semantically incomplete (e.g., pending operator runs older than an hour):

```bash
sqlite3 ./.ideation/coffee-shop-q2/logbook.sqlite \
  "SELECT operator_run_id, operator_name, started_at
   FROM operator_runs
   WHERE status = 'pending'
     AND datetime(started_at) < datetime('now', '-1 hour');"
```

If any rows appear, a prior session crashed mid-operator. Remediation: operator subagent logs are lost; the orchestrator patches these to `status='failed'` with `error='stale-pending'` on the next session start.

## Actions

The `decide.*` operator family is the bridge from logbook state to external artifacts.

### decide.shortlist(n)

- **Purpose:** mark the top `n` ideas as shortlisted for further work.
- **Readiness check:** at least `n` ideas with `status='active' AND score_summary IS NOT NULL`.
- **Dry-run:** `ideation_db.py query coffee-shop-q2 top-by-composite --n 10` prints the IDs that would be patched.
- **Effect:** patches `ideas.status='shortlisted'` for the selected IDs; no external system is touched.
- **Patch-back:** none — this action modifies the logbook itself, not an external target.

### decide.compare

- **Purpose:** render a side-by-side comparison of a cohort for human review.
- **Readiness check:** cohort non-empty; at least one metric has an assessment across the cohort.
- **Dry-run:** print cohort IDs and the metrics that will appear as columns; do not write the file.
- **Effect:** writes `./.ideation/{slug}/reports/<run_id>-compare.md` — a markdown table with one row per idea, columns for `title` + key latest-per-metric assessments + current `evidence_state`.
- **Patch-back:** none — the report is an export-only projection.

### decide.converge

- **Purpose:** final decision ceremony — promote 1–3 ideas to `status='selected'` and record rejected candidates.
- **Readiness check:** criteria_lock checkpoint was approved in this session's plan; `evaluate.score` has run on the cohort.
- **Dry-run:** print cohort with scores, stress-test verdicts, and tentative selections.
- **Effect:** patches `ideas.status` to `selected` or `rejected` per user's decision; writes `./.ideation/{slug}/reports/<run_id>-converge.md` capturing rationale.
- **Patch-back:** none (`ideas.status` is internal state, not an external identifier).

### decide.export(format)

- **Purpose:** produce a portable final artifact for sharing outside the logbook.
- **Readiness check:** at least one `status='selected'` or `status='shortlisted'` idea.
- **Dry-run:** print count of ideas that will be included and the target filename.
- **Effect:** writes `./.ideation/{slug}/reports/<run_id>-<format>.md` — markdown suitable for pasting into docs, issue trackers, or strategy memos. Supported formats: `menu` (Quick Wins / Core Bets / Moonshots buckets), `narrative` (prose summary), `table` (flat table).
- **Patch-back:** none.

### No cross-system commits

Unlike pre-tracker backlog logbooks, the ideation logbook does not have a `commit-to-jira` or `apply-to-miro` action at v1. Ideas that become tracked work exit the logbook via `decide.export` (human copies the artifact into the tracker). A future `commit-to-jira` action may be added for ideas with `status='selected'`; until then, no patch-back column exists for that purpose.

## Governance

- **Access:** single-owner per topic (recorded in `topic_meta.owner`). Any agent invoking the `ideation` skill in the containing project may append; read access is effectively anyone with filesystem access to the topic directory.
- **Lifetime:** indefinite per topic. Topics are created explicitly by the user and persist until the user deletes the topic directory.
- **Conflict resolution:** SQLite WAL mode is enabled at init. First-pass assumption is one `ideation` session per topic at a time; the skill does not detect concurrent sessions, but `operator_runs.status='pending'` older than 1 hour is the signal that a prior session crashed (see Validation / Incomplete-row scan).
- **Sunset rule:** manual. When a topic is concluded, the user may archive it (`mv ./.ideation/<slug> ./.ideation/_archive/<slug>`) or delete it. The skill itself does not auto-archive.
- **Gitignore:** `./.ideation/` is gitignored by default — ideation state is often half-formed. `decide.export` produces committable markdown artifacts when the user wants outputs shared. To share the raw logbook, users explicitly move or symlink the topic directory out of `./.ideation/`.
- **Schema versioning:** `schema_version: 1` in the YAML frontmatter. Schema changes require a migration script (added to `ideation_db.py`) that detects older versions by reading `_meta` or by probing for missing tables, then applies additive-only DDL where possible. Breaking changes bump the version and are gated behind explicit user opt-in.
